# 校园活动策划与报名管理平台 软件需求规格说明书

## 1. 项目背景
本文件是 `Requirements Agent` 的正式上游输入之一，用于描述最终要实现的产品目标，而不是对当前 Demo 的复述。

## 2. 产品目标
实现一个支持多角色协作的校园活动管理系统，覆盖活动全生命周期管理，并能在课程项目范围内落地为可演示、可扩展的完整 Web 产品。

## 3. 用户与场景
### 3.1 目标用户
- 游客
- 学生用户
- 活动负责人
- 管理员

### 3.2 核心场景
- 用户注册并登录系统。
- 学生或游客浏览已发布活动。
- 活动负责人创建活动草稿并完善信息。
- 活动负责人提交审核。
- 管理员审核活动。
- 审核通过后活动进入已发布状态并开放报名。
- 学生报名活动，系统校验名额、重复报名和报名资格。
- 活动负责人查看报名名单和人数。
- 管理员查看平台统计和审核记录。
- 学生注册并登录后，进入活动广场浏览近期活动。
- 学生根据“讲座”“志愿服务”“比赛”等类型筛选活动。
- 学生进入活动详情页，确认时间、地点、剩余名额和报名截止时间。
- 学生填写必要信息后完成报名。
- 学生在个人中心查看自己已报名的活动，并可在允许条件下取消报名。
- 活动负责人登录后进入自己的工作台。
- 创建一条新活动，填写完整信息后保存为草稿。
- 核对信息后提交审核。
- 若管理员驳回，则查看驳回原因并修改后重新提交。
- 审核通过后查看活动报名人数和名单。
- 管理员登录后台后查看待审核列表。
- 打开活动详情核验信息是否完整和合理。
- 对符合要求的活动执行通过操作。
- 对不符合要求的活动执行驳回并说明原因。
- 在后台查看活动总量、待审核数量和报名统计。
- 游客进入首页。
- 浏览已发布活动列表。
- 通过搜索和筛选缩小范围。
- 进入活动详情页查看完整信息。
- 若尝试报名，则引导登录。
- 学生进入注册页填写必要信息。
- 系统校验信息合法性并完成注册。
- 学生登录成功后进入学生视图。
- 学生在活动详情页点击报名。
- 系统校验活动状态、剩余名额、是否重复报名、是否已截止。
- 校验通过则创建报名记录。
- 系统更新已报名人数。
- 页面反馈报名成功。
- 学生在个人中心或活动详情页点击取消报名。
- 系统校验当前是否允许取消。
- 若允许，则删除或失效该报名记录。
- 系统回收名额并更新人数。
- 活动负责人进入工作台。
- 点击创建活动。
- 填写表单并保存草稿或直接提交审核。
- 提交成功后活动状态变为 `pending_review`。
- 管理员打开待审核活动。
- 核查关键信息。
- 点击审核通过。
- 活动状态更新为 `published`。
- 活动在活动广场展示，并开放报名。
- 发现信息缺失或内容不合理。
- 填写驳回原因并执行驳回。
- 活动状态更新为 `rejected`。
- 活动负责人在工作台查看驳回原因并修改后重新提交。
- 游客尝试报名
- 用户重复报名同一活动
- 活动满员后继续报名
- 报名截止后继续报名
- 非管理员访问审核中心
- 活动负责人提交信息不完整的活动
- 管理员驳回活动
- 学生取消已报名活动
- 已发布活动修改关键字段
- 活动被取消后学生查看详情

## 4. 功能需求
### FR-001 用户注册
- 描述：系统应支持用户注册，并确保相关角色可按规范完成对应操作。
- 角色：游客, 学生用户, 活动负责人, 管理员
- 前置条件：用户具备访问平台的基础条件
- 主流程：进入注册页面；填写必要信息；提交注册表单；系统校验后创建账号
- 备选流程：若账号标识已存在，则系统提示不可重复注册
- 后置条件：用户账号被创建
- 优先级：high
- 来源：brief:校园活动策划与报名管理平台-product-brief-7-功能范围-7-1-本期必须覆盖的功能范围-a-账户与身份
- 验收标准：AC-001

### FR-002 用户登录
- 描述：系统应支持用户登录，并确保相关角色可按规范完成对应操作。
- 角色：游客, 学生用户, 活动负责人, 管理员
- 前置条件：用户已拥有有效账号
- 主流程：进入登录页面；输入账号与密码；提交登录请求；系统验证身份并跳转
- 备选流程：若账号或密码错误，则系统给出明确错误提示
- 后置条件：用户进入与角色匹配的首页或工作台
- 优先级：high

## 6. 业务规则
- BR-001 游客只能浏览公开活动信息。
- BR-002 学生用户可以报名已发布活动，但不能审核活动。
- BR-003 活动负责人可以管理自己创建的活动，但不能审核自己的活动。
- BR-004 管理员可以审核和发布活动，但不能代替其他用户报名活动。
- BR-005 `draft`：草稿，活动负责人可继续编辑。
- BR-006 `pending_review`：已提交审核，等待管理员处理。
- BR-007 `rejected`：审核未通过，活动负责人可查看驳回原因并修改后重新提交。
- BR-008 `published`：审核通过并对学生开放报名。
- BR-009 `registration_closed`：活动仍保留展示，但不再允许报名。
- BR-010 `completed`：活动已结束。
- BR-011 `cancelled`：活动被取消。
- BR-012 只有 `published` 状态的活动允许报名。
- BR-013 当活动报名人数达到上限时，系统不得再接受新的报名请求。
- BR-014 同一学生不能重复报名同一活动。
- BR-015 学生在活动报名截止前可以取消报名，取消后名额应释放。
- BR-016 已结束、已取消、未发布的活动不能报名。
- BR-017 活动名称
- BR-018 活动类型
- BR-019 活动简介
- BR-020 主办方
- BR-021 活动地点
- BR-022 活动开始时间
- BR-023 活动结束时间
- BR-024 报名截止时间
- BR-025 人数上限
- BR-026 面向对象/参与范围
- BR-027 信息不完整的活动不能审核通过。
- BR-028 管理员驳回活动时必须填写驳回原因。
- BR-029 已审核通过的活动如需修改关键字段，应重新进入审核流程。
- BR-030 审核记录应保留审核人、审核时间、审核结论和说明。
